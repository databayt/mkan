# Story 3.1: E2E job in CI

> **Epic**: 3 · **Status**: To Do · **Phase**: B

## Story
As a release engineer, I want Playwright E2E to run on every PR, so that user-facing regressions are caught before merge.

## Acceptance Criteria
1. New `e2e` job in `.github/workflows/ci.yml`.
2. Steps: install Playwright browsers, run `pnpm test:e2e`, upload report on failure.
3. Job runs against a seeded test DB (use Neon test branch or in-job postgres).
4. Sharded into 2 parallel jobs (matrix `shardIndex: [1,2]`, `shardTotal: 2`).
5. CI runtime ≤ 12 min total.
6. Job blocks merge.

## Tasks
- [ ] Add `e2e` job to `ci.yml`.
- [ ] Add `playwright install --with-deps chromium`.
- [ ] Configure `playwright.config.ts` to read `--shard` from env or args.
- [ ] Set `BASE_URL` for the job; let Playwright start `next dev` if unset.
- [ ] Provision `DATABASE_URL_TEST`; seed before run.

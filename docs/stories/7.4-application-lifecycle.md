# Story 7.4: Application lifecycle

> **Epic**: 7 · **Status**: To Do · **Phase**: C

## Story
As a tenant, I want to submit an application; as a manager, I want to review and approve/deny it; on approval, a Lease is created.

## Acceptance Criteria
1. `createApplication(input)` action — tenant submits.
2. `approveApplication(id)` — creates Lease (start/end dates, monthly rent), sends email.
3. `denyApplication(id, reason)` — marks denied, emails tenant.
4. UI: tenant sees status; manager has approve/deny buttons.

# Story 4.10: Admin payment verification queue

> **Epic**: 4 · **Status**: To Do · **Phase**: C

## Story
As an admin, I want a queue of pending reference-based payments showing reference + receipt + booking, so that I can verify each one and confirm bookings.

## Acceptance Criteria
1. `/admin/payments` has tab "Pending verification".
2. Lists `PaymentReference` rows with `verifiedAt = null`.
3. Each row: gateway, amount, reference, receipt URL preview, booking link.
4. Verify button → `verifyPaymentReference({ id })` → marks `verifiedAt`, sets Payment status `PAID`, calls `confirmBooking`, sends email.
5. Reject button → marks Payment as `FAILED`, sends rejection email, frees seats.

## Files
- `src/app/[lang]/admin/payments/page.tsx`, action extension.

## Tests
- Unit: verify and reject paths.
- E2E: admin verifies one ref → booking shows confirmed.

## Definition of Done
- [ ] AC met · tests green

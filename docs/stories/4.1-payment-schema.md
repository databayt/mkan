# Story 4.1: PaymentGateway enum + PaymentReference model

> **Epic**: 4 · **Status**: To Do · **Phase**: C

## Story
As a developer, I want a unified payment data model that supports all gateways (Stripe + Bankak + Cashi + mobile money + bank transfer + cash), so that subsequent payment stories have a stable foundation.

## Acceptance Criteria
1. `PaymentGateway` enum added: `STRIPE`, `BANKAK`, `CASHI`, `MOBILE_MONEY`, `BANK_TRANSFER`, `CASH`.
2. `Payment` model extended with `gateway: PaymentGateway`, `gatewayPaymentId: String?`, `pendingVerification: Boolean @default(false)`.
3. New `PaymentReference` model: `id`, `paymentId`, `gateway`, `reference` (txn ref), `verifiedAt: DateTime?`, `verifiedBy: User?`, `receiptUrl: String?`.
4. Same fields available for `TransportPayment` (or unified into `Payment`).
5. Migration committed and applied.
6. tsc clean post-`prisma generate`.

## Files
- `prisma/schema.prisma`, new migration under `prisma/migrations/`.

## Definition of Done
- [ ] AC met · migration applied to dev DB · prisma generate clean

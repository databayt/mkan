# Story 1.2: Zod env validation at boot

> **Epic**: 1 · **Status**: To Do · **Phase**: B

## Story
As a developer, I want `process.env` validated by a single zod schema at module load, so that missing/malformed env vars fail fast at boot instead of leaking as runtime errors.

## Acceptance Criteria
1. `src/lib/env.ts` exports `env` — a strongly typed object validated by zod.
2. Required vars (`DATABASE_URL`, `AUTH_SECRET`, `NEXTAUTH_URL`, `IMAGEKIT_PRIVATE_KEY`, `NEXT_PUBLIC_IMAGEKIT_PUBLIC_KEY`, `NEXT_PUBLIC_IMAGEKIT_URL_ENDPOINT`, `RESEND_API_KEY`, `EMAIL_FROM`, `UPSTASH_REDIS_REST_URL`, `UPSTASH_REDIS_REST_TOKEN`, `STRIPE_SECRET_KEY`, `STRIPE_WEBHOOK_SECRET`) and OAuth vars (Google, Facebook) are validated.
3. Build fails with a readable error if any required var is missing.
4. Server-only vars are not exposed to client.

## Tasks
- [ ] Create `src/lib/env.ts` with zod schema split into `serverEnvSchema` and `clientEnvSchema`.
- [ ] Throw on parse error with each missing key listed.
- [ ] Replace ad-hoc `process.env.X` reads in actions where convenient.

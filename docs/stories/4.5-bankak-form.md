# Story 4.5: Bankak reference-based form

> **Epic**: 4 · **Status**: To Do · **Phase**: C

## Story
As a guest paying via Bankak, I want to see the mkan Bankak account, pay in my own app, then enter the transaction reference, so that admin can verify and confirm my booking.

## Acceptance Criteria
1. `src/components/booking/payment/bankak-form.tsx` shows: payee account, amount in SDG, reference input, submit.
2. On submit, calls `submitPaymentReference({ bookingId, gateway: 'BANKAK', reference })`.
3. Action creates `Payment` (status = `PENDING_VERIFICATION`) and `PaymentReference` row.
4. UI shows "Awaiting verification" state with admin SLA notice.

## Files
- new component, `src/lib/actions/payment-actions.ts` (extend with `submitPaymentReference`).

## Tests
- Unit: action validation (unique ref per gateway).
- Component: form renders, validates, submits.

## Definition of Done
- [ ] AC met · tests green

# Story 4.3: Stripe webhook handler

> **Epic**: 4 · **Status**: To Do · **Phase**: C

## Story
As a system, I want to listen for Stripe events (`payment_intent.succeeded`, `payment_intent.payment_failed`, `charge.refunded`), so that bookings auto-confirm/fail/refund on Stripe state.

## Acceptance Criteria
1. New route `src/app/api/webhooks/stripe/route.ts` (`runtime = "nodejs"`).
2. Verifies signature with `stripe.webhooks.constructEvent` and `STRIPE_WEBHOOK_SECRET`.
3. Dispatches by event type → updates Payment + Booking → sends email.
4. Idempotent: re-delivery of same event does not double-confirm.
5. Replaces TODO at `payment-actions.ts:650`.

# Story 4.12: Unit tests for payment actions

> **Epic**: 4 · **Status**: To Do · **Phase**: C

## Story
As a developer, I want unit tests covering every gateway action and edge case, so that we catch payment regressions in CI.

## Acceptance Criteria
1. `tests/actions/payment-actions.test.ts` extended to cover:
   - `createStripePaymentIntent` (success, unauthenticated, invalid booking, currency mismatch).
   - `handleStripeWebhook` (signature valid/invalid, success/fail/refund events, idempotency).
   - `processRefund` (full, partial, already refunded, Stripe API error).
   - `submitPaymentReference` (each gateway, duplicate ref, invalid ref).
   - `verifyPaymentReference` (success, already verified, missing ref).
2. Stripe SDK mocked.
3. Coverage > 90% for `payment-actions.ts`.
